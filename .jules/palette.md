## 2024-05-15 - ARIA Toggles for Framer Motion Expandables
**Learning:** In React codebases utilizing `framer-motion` for expandable UI sections (like score breakdowns), it's a common pattern to conditionally render the `<motion.div>` entirely based on state (e.g., `showBreakdown && <motion.div>`) rather than toggling visibility via CSS. When this conditionally rendered element acts as the controlled content for a toggle button, it requires a static `id` that matches the button's `aria-controls` attribute, ensuring screen readers can correctly associate the two elements when the content mounts.
**Action:** Always verify that collapsible containers controlled by buttons have explicitly assigned `id`s and that the controlling button possesses matching `aria-controls` and dynamic `aria-expanded` attributes, especially when conditional rendering frameworks are used.
